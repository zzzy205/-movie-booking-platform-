# 电影预订平台 - 部署策略文档

## 项目概述
- **项目名称**: movie-booking-platform
- **部署环境**: Ubuntu服务器 + 内网环境
- **网络特点**: 开发阶段有互联网，生产阶段仅内网运行

## 部署流程

### 1. 开发阶段
- **开发环境**: 本地开发机器（有互联网连接）
- **代码管理**: 修改代码并上传到远程代码仓库
- **版本控制**: Git + 远程仓库

### 2. 部署阶段
- **部署环境**: Ubuntu服务器
- **代码获取**: 从远程仓库拉取最新代码
- **部署方式**: Docker容器化部署
- **服务启动**: 使用Docker Compose管理服务

### 3. 生产阶段
- **网络环境**: 服务器断开互联网，重新分配内网IP
- **服务访问**: 内网用户通过内网IP:端口号访问服务
- **网络配置**: 仅需配置端口号，无需考虑公网IP或内网穿透

## 技术架构

### 网络架构
```
内网用户 → 内网IP:端口 → Ubuntu服务器 → Docker容器 → 应用服务
```

### 技术选型
- **容器化**: Docker + Docker Compose
- **反向代理**: Nginx（端口映射和负载均衡）
- **配置管理**: 环境变量区分开发/生产环境
- **服务发现**: 内网IP + 固定端口号

### 端口配置策略
- 固定内网端口号（如80、443、3000等）
- 使用内网IP地址
- 配置防火墙规则
- 考虑负载均衡需求

## 部署流程优化

```
代码提交 → 远程仓库 → 服务器拉取 → Docker构建 → 服务启动 → 内网访问
```

## 关键考虑点

### 1. 网络配置
- 开发阶段：服务器有互联网，正常拉取代码
- 生产阶段：服务器无互联网，仅内网运行
- 使用内网IP + 端口号提供服务

### 2. 环境管理
- 区分开发环境和生产环境配置
- 使用环境变量管理网络配置
- Docker配置支持环境切换

### 3. 部署自动化
- 部署脚本自动化
- 服务健康检查
- 日志监控和管理
